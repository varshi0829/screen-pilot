# 3. Product Vision

## 3.1 Core Vision

ScreenPilot is GPS for software. Just as a GPS system knows the destination, monitors the current position, identifies the next turn, highlights it on the map, and recalculates when the user takes a wrong turn, ScreenPilot knows the goal, observes the current page, identifies the next action, highlights the target element, and adapts when the page changes.

## 3.2 Key Principles

1. **User remains in context**: The user never leaves the website they are using. The extension overlays the current page.
2. **Visual over conversational**: ScreenPilot highlights elements rather than describing steps in text.
3. **Closed-loop feedback**: Every action is observed. Every page change triggers re-analysis.
4. **Goal-oriented**: The system tracks the end state, not just the next step.
5. **Resilient to change**: When UI elements change, the system adapts rather than failing.

## 3.3 The Experience

The experience should feel like having a co-pilot who:

- Is looking over your shoulder
- Understands what you want to accomplish
- Points to the next element and says "click here"
- Watches what happens
- Adjusts based on the result
